不是文生图，是可控的语音绘图系统

Say2Draw 的核心是“系统谦逊”：AI 不假装一次画完，而是先给出可编辑的草图起点，让用户通过语音不断修正。

纯语音闭环

自动持续监听、实时转写、语音反馈和回声保护，绘图操作不依赖鼠标键盘。

计划确认机制

复杂指令先生成绘图计划，用户说“确认”后执行，降低误识别带来的误操作。

手绘笔触管线

LLM 输出结构化笔触坐标，前端用 Bézier 曲线拟合，稳定生成草图质感。

局部可编辑

支持撤销、恢复、移动、缩放、改色和视觉编辑，每一步都能累积在画布上。

SketchAgent：让 LLM 负责语义，让代码负责线条

项目的关键不是把 prompt 扔给图片生成模型，而是把自然语言转换成可审计、可恢复、可继续编辑的结构化绘图动作。 这条管线参考 SketchAgent 论文中的笔触表示与曲线拟合思路，并在 PR17 中落地为 50×50 网格坐标到 Bézier/SVG 的渲染流程。

1

语音识别与审查

Web Speech API 产出实时转写，并检测“换/话/化”等疑似误识别词。

2

本地解析优先

清空、撤销、移动、缩放、改色等指令直接本地执行，减少 LLM 成本。

3

LLM 计划与笔触 XML

复杂场景由模型拆解为计划或 50×50 网格上的结构化笔触。

4

Bézier 拟合渲染

前端把坐标采样点拟合为平滑 SVG path，叠加到 Canvas 工作区。

技术栈

项目用浏览器原生语音能力连接 React 工作台、Express LLM 服务和 Canvas/SVG 渲染层，形成一条可运行、可解释、可继续迭代的语音绘图链路。

前端

React 19 + TypeScript + Vite 8 构建工作台，用 useReducer 管理绘图状态、历史记录和语音反馈。

后端

Express.js + OpenAI SDK 提供 OpenAI-compatible API 代理、模型配置接口和结构化绘图计划生成。

语音与绘图

Web Speech API 负责 SpeechRecognition 和 SpeechSynthesis，HTML5 Canvas 承载画布，SVG overlay 呈现手绘笔触。

AI 与渲染

Mimo 多模态默认模型，兼容 OpenAI-compatible provider；SketchAgent 50×50 网格输出经 Bézier 曲线拟合渲染。

和 AI 文生图的根本区别

Say2Draw 追求的是可控表达，而不是一次性生成一张不可拆开的图片。

维度 AI 文生图 Say2Draw
执行单位 整张图片 结构化图形与笔触
迭代方式 重新生成整图 增量添加、局部修改、可撤销恢复
用户控制 靠 prompt 试错 自然语音逐轮指挥
响应速度 等待模型生成 本地动作实时执行，复杂动作按需调用 LLM
