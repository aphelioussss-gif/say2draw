# PLAN - Say2Draw 开发计划

## 1. 开发策略

### 1.1 分支策略

- `main`：始终可运行，只接受 PR 合并
- `feature/*`：每个功能独立分支
- PR 必须通过基础检查才能合并

### 1.2 提交规范

```
feat: 新功能
fix: 修复
docs: 文档
chore: 构建/工具
refactor: 重构
test: 测试
```
