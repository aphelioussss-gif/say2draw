# Say2Draw 产品定位说明

## 核心定位

Say2Draw 是一个语音驱动的白板草图工具。它把自然语言转成可编辑的流程图、教学图和概念示意图，让用户不用拖拽、不用选工具，就能快速把想法放到画布上。

当前 MVP 优先支持结构化表达，复杂插画能力仍在探索中。

## 解决的问题

Say2Draw 解决的问题是：用户脑子里有一个结构，但不想打开复杂绘图工具，也不想手动拖框、连线、排版。

典型用户包括：

- 老师
- 产品经理
- 学生
- 会议记录者
- 需要快速解释方案的人

这些用户经常需要表达：

- 一个流程
- 一个关系
- 一个概念图
- 一个课堂示意
- 一个方案结构

传统工具的交互成本很高：

- 选形状
- 拖位置
- 连箭头
- 改文字
- 调大小
- 对齐

Say2Draw 希望把这些操作变成自然语言：

- “画一个登录到支付的流程图”
- “把支付节点往右一点”
- “加一个失败重试分支”
- “把整体放大一点”

这比“画一只猫”更有产品价值。

## 模板的意义

模板不是产品卖点，模板是底层质量保障。

模板的作用是让结果稳定：

- 流程图用确定性布局
- 太阳、地球、月亮用教学图模板
- 常见对象用对象模板
- 本地调整用确定性变换

用户感知到的不是模板，而是：

> 我说一句话，系统理解了，并且生成了一个能继续改的图。

这才是产品体验。

## 不做什么

Say2Draw 当前不应主打“AI 画任意图”。

原因很简单：如果用户期待的是精美图像，Midjourney、DALL·E、即梦、可灵这类产品更强。Say2Draw 不适合和它们比画质。

小猫、人物、复杂场景都可以作为边缘能力。它们不好看，不影响主线。

## 路演主线

路演应当围绕“语音控制的结构化白板”展开。

对应功能重点：

1. 语音输入
2. 计划确认
3. 结构化图生成
4. 局部语音修改
5. 累积添加
6. 撤销恢复

推荐演示路径：

1. “画一个从语音输入到生成草图的流程图”
2. “确认”
3. “把整体放大一点”
4. “往右一点”
5. “加一个反馈节点”
6. “撤销”

## 一句话总结

Say2Draw 的意义不在替用户画画，而在让用户用说话完成白板表达。
