# Say2Draw｜以声绘色

> 纯语音控制的 AI 绘图工具。说你想画的，系统听懂、计划、确认、执行，草图画出来你继续用语音打磨。

## 项目简介

Say2Draw 是一款纯语音控制的 AI 绘图工具。用户只需用自然语言说出绘图意图，系统完成语音识别、意图理解、绘图计划生成，经用户确认后在 Canvas 上渲染为几何图形拼贴画或手绘笔触风格的草图。所有操作——从描绘画布内容到调整布局到撤销恢复——均通过语音完成，无需鼠标或键盘参与绘图。

Say2Draw 的核心理念是**系统谦逊**：AI 不替用户一次性"画好"，而是从保守草图起步，让用户用自然语言逐轮打磨。技术上，这一理念通过 **SketchAgent 笔触管线**实现：LLM 输出 50×50 网格上的结构化笔触坐标（而非像素位图），前端通过最小二乘 Bézier 曲线拟合将其渲染为手绘风格的 SVG 笔触。LLM 负责"画什么"的语义理解，确定性渲染负责"怎么画"的视觉品质——两者分工，各自在擅长领域做到极致。

不是"AI 替你画好"的文生图，而是"你指挥 AI，草图起步，一起打磨"。

## 与 AI 文生图的区别

| 维度 | AI 文生图 (Midjourney/DALL·E) | Say2Draw |
|---|---|---|
| 执行单位 | 整张图片 | 结构化图形 |
| 可编辑性 | 黑箱，无法修改局部 | 结构化，可逐个元素调整 |
| 响应方式 | 等待生成（数秒到数十秒） | 实时执行 |
| 迭代成本 | 重新生成整张图 | 语音修改单个元素 |
| 累积性 | 每次独立生成 | 图形持续累积在画布上 |
| 用户控制 | 低（prompt 试错） | 高（逐轮语音微调） |

## 功能特性

### 语音交互
- 🎤 页面加载后自动开始持续监听，无需点击"开始"
- 📝 实时转写（interim）和最终识别（final）双轨展示
- 🔍 语音转录审查门：自动检测"换""话""化"等疑似误识别词
- 🛡️ 回声保护：系统播报期间忽略自身语音
- ⏯️ 紧急暂停/恢复监听控制

### 绘图模式
- 🎨 **自由绘画**：随手涂鸦，适合快速记录灵感
- 🌙 **故事场景**：叙事性画面，有情绪和故事感
- 📐 **课堂讲解**：教学图示，清晰标注元素关系
- 🔄 **白板流程**：流程图、架构图、步骤示意

### 绘图能力
- 📋 计划→确认→执行三阶段流程：系统先生成绘图计划，用户确认后再执行
- ✏️ 手绘笔触风格草图渲染（Bezier 曲线拟合）
- 📍 9 区空间模型：支持"在画布左上角画..."等空间指令
- 🚫 防重叠自动分布：多个对象自动避免堆叠
- 🔍 视觉编辑：发送当前画布截图给 LLM，用语音描述修改
- ➕ 累积添加：在已有草图上增量添加新元素
- 🔄 流程图结构化编辑：LLM 输出结构化意图，确定性布局器重排
- ↩️ 撤销/恢复：支持回到上一版快照
- ⚡ 本地即时调整：放大、缩小、往右移、改颜色等无需 LLM 调用

### 反馈系统
- 🗣️ SpeechSynthesis 语音反馈
- 📊 四段式反馈面板（PR25）：我听到 | 我理解为 | 执行结果 | 建议
- 🏷️ 五态状态指示：空闲、聆听中、思考中、成功、失败
- 💡 失败场景给出可直接复述的建议说法

### LLM 配置
- 🔧 内建设置页：配置 API Key、Base URL、模型名称
- 🔀 供应商无关：OpenAI-compatible 接口，支持 Mimo 等多模态模型
- 💰 成本控制：本地解析优先，LLM 仅用于复杂指令和视觉编辑

## 架构概览

```
用户语音 → Web Speech API → 实时转写 → 转录审查门
                                          ↓
                              ┌─ 本地解析（颜色/形状/撤销/清空）
                              │
              指令路由 ────────┼─ LLM 意图识别 + 计划生成
                              │
                              └─ 视觉编辑（流程图/草图调整）
                                          ↓
                              绘图计划 → 用户确认 → 草图生成
                                          ↓
                   ┌──────────────────────┘
                   ↓
         SketchAgent 笔触管线
    LLM 输出 50×50 网格笔触坐标
              ↓
    XML 解析（sketchParser）
              ↓
    最小二乘 Bézier 曲线拟合（bezierFitter）
              ↓
    网格→像素映射（svgRenderer）
              ↓
    SVG 笔触渲染（SketchLayer）
              ↓
         语音反馈
```

### SketchAgent 笔触管线

Say2Draw 的绘图质量来源于 PR17 引入的 SketchAgent 统一笔触管线。它不是让 LLM 直接输出像素坐标或位图，而是将绘图过程拆成两个正交的职责：

| 阶段 | 职责 | 执行者 |
|---|---|---|
| 语义理解 | 决定画什么、在哪画、什么形状 | LLM（Mimo 多模态模型） |
| 笔触渲染 | 将结构描述转为有手绘质感的线条 | 确定性代码（Bézier 拟合 + SVG） |

具体流程：
1. **LLM 生成笔触 XML**：系统 Prompt 教 LLM 用 50×50 编号网格描述每一笔的坐标序列和时序参数（t_values）
2. **XML 解析**：从 LLM 返回文本中提取 `<strokes>` 块，处理 Markdown code fence、坐标越界等容错
3. **Bézier 曲线拟合**：对每笔的采样点用最小二乘法求解三次 Bézier 控制点，使曲线逼近原始点集
4. **网格→像素映射**：50×50 逻辑坐标系等比映射到 800×500 Canvas 物理画布
5. **SVG 渲染**：拟合后的平滑曲线作为 SVG path 叠加在 Canvas 之上

这套管线让 LLM 的"手绘感"得以稳定复现：线条不会机械平直，转角自然圆润，草图质感一致。同时，因为笔触描述是结构化的（坐标数组），后续的局部编辑、布局调整、快照恢复都有了可操作的锚点。

## 快速开始

### 环境要求

- Node.js >= 18
- 现代浏览器（推荐 Chrome，需要 Web Speech API 支持）

### 安装与运行

```bash
npm install
npm run dev          # 前端开发服务器
```

默认情况下，本地解析和语音识别即可工作。如需 LLM 增强功能：

```bash
cp .env.example .env
# 编辑 .env 填入 LLM_API_KEY、LLM_BASE_URL、LLM_MODEL
npm run server       # 后端 API 服务
npm run dev          # 前端（另一个终端）
```

或在应用内设置页直接配置 LLM。

### 构建

```bash
npm run build
```

### 基础绘图

| "画一个红色圆形" | 基础几何图形 |
| "画一个蓝色矩形" | 基础几何图形 |
| "画一条黑色线" | 基础几何图形 |
| "写上你好" | 画布文字 |
| "清空画布" | 清除全部 |
| "撤销" | 回退上一步 |
| "恢复上一版" / "撤回刚才" | 恢复到上一个快照 |

### 复杂场景
| 指令示例 | 说明 |
|---|---|
| "画一个小女孩站在月亮下面" | 故事场景模式 |
| "画一个从登录到支付的流程图" | 白板流程模式 |
| "画太阳、地球和月亮，表示月亮绕地球转" | 课堂讲解模式 |
| "画一只猫蹲在窗台上" | 自由绘画模式 |

### 草图打磨
| 指令示例 | 说明 |
|---|---|
| "往右一点" / "放大" / "缩小" | 本地即时调整（无需 LLM） |
| "改颜色为蓝色" | 本地即时调整 |
| "加一点细节" / "轮廓更清楚" | LLM 视觉编辑 |
| "宽松" / "太窄了" / "散开一点" | 流程图布局调整 |
| "就这样" / "可以了" | 确认草图退出打磨模式 |
| "取消" / "算了" / "不画了" | 放弃当前操作 |

### 确认交互
| 指令示例 | 说明 |
|---|---|
