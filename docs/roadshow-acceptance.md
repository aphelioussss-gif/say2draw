# Roadshow Acceptance Checklist

> 路演前验收清单。每条需手动确认通过后，方可进入路演流程。

## 1. 流程图生成

| 测试项 | 指令 | 通过标准 |
|---|---|---|
| 流程图生成 | 画一个用户登录到订单支付的流程图 | 出现 3-6 个节点，箭头方向正确，节点文字可读 |
| 流程图二次确认 | 画 Say2Draw 的工作流程 | 出现不同节点内容，结构不重复 |

## 2. 计划确认

| 测试项 | 指令 | 通过标准 |
|---|---|---|
| 确认绘制 | 确认 | plan 转为可见图形 |
| 取消重来 | 取消 | plan 关闭，画布无变化 |

## 3. 本地调整

| 测试项 | 指令 | 通过标准 |
|---|---|---|
| 放大 | 把整体放大一点 | 图形整体放大且不出界 |
| 缩小 | 把整体缩小一点 | 图形整体缩小且仍可读 |
| 右移 | 往右一点 | 图形整体右移，位置合理 |
| 左移 | 往左一点 | 图形整体左移，位置合理 |
| 上移 | 往上一点 | 图形整体上移，位置合理 |
| 下移 | 往下一点 | 图形整体下移，位置合理 |

## 4. 流程编辑

| 测试项 | 指令 | 通过标准 |
|---|---|---|
| 添加节点 | 加一个失败重试节点 | 新节点出现，流程结构可读 |

## 5. 撤销

| 测试项 | 指令 | 通过标准 |
|---|---|---|
| 撤销 | 撤销 | 回到上一版 |
| 多次撤销 | 撤销、撤销 | 连续回退，不崩溃 |

## 6. 日志系统

| 测试项 | 指令 | 通过标准 |
|---|---|---|
| 日志面板可见 | 点击日志面板按钮 | 面板展开，显示日志列表 |
| 日志记录 | 执行任意操作后查看日志 | 每条操作均有对应日志条目 |
| 失败日志 | 手动触发失败场景 | 日志显示 fallback 或 error 状态 |
| 复制日志 | 点击复制按钮 | 日志内容粘贴到剪贴板 |
| 清空日志 | 点击清空按钮 | 日志列表被清空 |
