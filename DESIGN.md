# DESIGN.md - Say2Draw 设计文档

## 1. 设计目标

创建一个直观、响应迅速的语音绘图体验，让用户专注于创意表达而非工具操作。

## 2. 竞品分析

### 2.1 现有方案分类

**生成式 AI 方案**

| 产品 | 流程 | 优点 | 缺点 |
|------|------|------|------|
| ChatGPT/DALL-E 3 | 语音→文本→生成图片 | 简单，效果好 | 黑箱，无法修改局部 |
| Midjourney | 同上 | 高质量艺术风格 | 同上 |
| Adobe Firefly | 同上 | 集成设计工具链 | 同上 |
| Canva Magic Media | 同上 | 易用 | 同上 |

**研究项目方案**

| 项目 | 特点 | 局限 |
|------|------|------|
| VoCreate / VoiceDraw | 结构化命令，针对运动障碍用户 | 研究项目，非商业产品 |
| ARTiculate | 免手绘图，语音控制画笔移动 | 命令复杂，学习成本高 |
| TalkSketch | 自由绘图+实时语音 | 嵌入式，非 Web 应用 |

### 2.2 Say2Draw 定位

| 维度 | 生成式 AI | 研究项目 | Say2Draw |
|------|-----------|----------|----------|
| 目标用户 | 普通用户 | 障碍用户 | 想快速表达想法的人 |
| 交互方式 | 文本 prompt | 复杂命令 | 自然语言 |
| 执行模型 | 整张图生成 | 逐笔画 | 结构化 action |
| 可控性 | 低 | 高 | 高 |
| 学习成本 | 低 | 高 | 低 |
| 实时性 | 等待生成 | 实时 | 实时 |

### 2.3 差异化总结

Say2Draw = 自然语言的低门槛 + 结构化绘图的高可控性

具体差异化点：

1. 累积式绘图：每一步都在画布上留下痕迹
2. 可中断：说到一半可以停，不用等它画完
3. 可局部修改：可以只改某个图形，而不是重画整张图
4. 自然语言：不需要学复杂命令
5. Web 应用：浏览器直接用，不用装软件

## 3. 交互设计

### 3.1 语音交互流程

```
用户说话 → 语音识别 → 文本转写 → LLM 解析 → 绘图指令 → Canvas 执行
```

### 3.2 反馈机制

- **视觉反馈**：语音识别状态指示器
- **听觉反馈**：操作确认音效
- **画布反馈**：实时绘图预览

### 3.3 自动持续监听取舍

Say2Draw 不把"点击开始监听"作为核心路径。页面加载后会自动尝试启动 Web Speech API，并通过状态区提示用户是否正在监听、是否需要麦克风权限、浏览器是否支持。

浏览器弹出的麦克风授权属于安全机制，不视为产品内绘图操作。PR 4 只完成语音文本识别展示，不直接执行绘图，避免在 parser 完成前误触发 Canvas 操作。

PR 6 增加"暂停监听 / 恢复监听"安全控制。该控制只用于紧急停止麦克风监听，不承担绘图功能；正式绘图仍通过语音指令完成。

PR 7 增加语音反馈闭环。系统执行指令后会使用 SpeechSynthesis 播报结果，并在播报期间暂停或忽略 SpeechRecognition 的识别结果，避免系统把自己的反馈当成新的用户指令。

PR 8 修复语音反馈的边界问题：状态同步（异步回调中访问最新手动暂停状态）和并发处理（快速连续语音播报时回调不丢失）。

PR 9 增加 LLM 结构化指令解析器。本地解析失败后，系统可选调用 OpenAI-compatible API 解析复杂指令。LLM 使用 Structured Outputs 返回结构化 DrawingAction JSON，返回结果经过运行时校验，非法 action 不执行。未配置 API key 或 LLM 调用失败时，系统降级为 ask_clarification。
