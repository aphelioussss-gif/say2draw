# Roadshow Demo Script

> 路演演示脚本。全程按顺序执行，无需临场发挥。

## 主演示流程（7 步，建议 5-7 分钟）

### Step 1：生成流程图

**你说：**
```
画一个用户登录到订单支付的流程图
```

**系统行为：**
- 生成 6 节点流程图：登录 → 验证身份 → 选择商品 → 提交订单 → 支付 → 支付成功
- 显示绘图计划预览

**确认点：**
- 节点数量 3-6 个
- 箭头方向从左到右

---

### Step 2：确认计划

**你说：**
```
确认
```

**系统行为：**
- plan 转为画布上的流程图图形
- 包含方框节点和箭头连线

---

### Step 3：放大

**你说：**
```
把整体放大一点
```

**系统行为：**
- 流程图整体放大
- 不出画布边界

---

### Step 4：右移

**你说：**
```
往右一点
```

**系统行为：**
- 流程图整体右移
- 位置合理

---

### Step 5：添加节点

**你说：**
```
加一个失败重试节点
```

**系统行为：**
- 新节点出现，连接到原有流程
- 流程结构保持可读

---

### Step 6：撤销

**你说：**
```
撤销
```

**系统行为：**
- 回到添加节点前的版本
- 图形恢复原状

---

### Step 7：查看日志

**你说：**
```
（打开日志面板）
```

**系统行为：**
- 日志面板展示前述 6 步的完整记录
- 包含每条指令、阶段、状态、耗时

**示范话术：**
> 这里记录了刚才每一步的执行情况。可以看到每次请求的指令、耗时、和返回结果。如果有任何异常，日志会标出 warning 或 error，便于定位问题。

---

## 备用失败演示（1-2 分钟）

在日志面板已展示的前提下，用于说明系统的能力边界。

**你说：**
```
画一只小猫
```

**系统行为：**
- LLM 理解用户想画猫
- 本地模板库没有猫，走 generic outline 兜底
- 生成抽象轮廓

**示范话术：**
> 这个场景展示的是当前系统的能力边界。系统能正确理解用户意图，但本地模板库尚未覆盖猫的笔触模板，LLM 直接生成 50×50 坐标时比例不稳定，所以走了通用兜底。
>
> 日志面板中可以看到这一条标记为 fallback，类型为 no_template。这个失败不会影响路演主线，因为我们把演示集中在结构化白板场景——流程图、教学图、概念图。复杂自由绘画作为后续迭代方向。
